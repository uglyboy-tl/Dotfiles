# 执行计划

## 概述

加载计划 → 批判性审查 → 分批执行 → 批次间报告审查

**核心原则：** 分批执行，设置检查点。

**开始时声明：** "我正在使用 executing-plans 技能来实现此计划。"

## 执行流程

### 1. 加载并审查计划

**智能文件选择：**
1. 根据用户信息（功能名、日期、关键词）确定计划文件
2. 检查 `docs/plans/` 文件夹，匹配最相关文件
3. 决策：
   - 明确则直接加载并告知用户
   - 不明确或多个选项，列出让用户选择
   - 仅有一个文件，自动加载
4. 告知用户加载哪个文件

**审查流程：**
1. 读取计划文件
2. 批判性审查，识别问题、缺失、模糊指令
3. 有疑虑 → 执行前向用户提问
4. 无疑虑 → 创建待办清单并执行

### 2. 执行批次（默认 3 个任务）

逐任务执行：
1. 标记为进行中
2. 按计划步骤执行
3. 运行指定验证
4. 验证通过 → 标记已完成

### 3. 批次报告

完成后：
- 展示已完成任务和产出
- 提供验证输出和检查结果
- 说："准备接收反馈。"

### 4. 继续执行

根据用户反馈：
- 应用修改
- 执行下一批
- 重复直到完成

### 5. 完成任务

- 移动计划文档：`docs/plans/` → `docs/completed/`
- 报告完成情况，提供执行总结

## 何时停止执行

**立即停止当前批次：**
- 遇到阻碍：缺少依赖、检查失败、指令不明确
- 计划存在关键缺陷无法开始
- 不理解指令或执行步骤
- 验证反复失败（2-3 次后仍失败）

**停止后操作：**
1. 立即停止，不猜测或绕过
2. 具体描述问题及上下文
3. 请求用户澄清，等待指令

## 何时重新审查计划

**返回步骤 1 的情况：**
- 用户更新了计划文件
- 执行中发现方法需重新思考
- 发现计划结构性错误或矛盾

**原则：** 遇到阻碍不强行突破，质量优先于速度。

## 记住

- 首先批判性审查计划
- 严格按照计划步骤执行
- 不跳过验证步骤
- 计划提到时引用相关技能
- 批次之间：仅报告并等待
- 遇到阻碍时停止，不猜测
