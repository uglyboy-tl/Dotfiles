# TDD 命令

使用严格的测试驱动开发实现以下功能：$ARGUMENTS

## 核心要求

- **测试先行**：每个功能必须从编写失败测试开始
- **最小实现**：编写刚好足够的代码使测试通过
- **持续重构**：在测试保持绿色的前提下改进代码
- **覆盖率达标**：确保测试覆盖率不低于 80%，关键逻辑 100%

## 关键检查点

1. **接口定义**：明确输入/输出契约
2. **失败测试**：编写测试并验证其失败
3. **通过实现**：编写最小化代码使测试通过
4. **重构优化**：消除重复，改进命名
5. **覆盖率验证**：运行覆盖率报告，补充测试

## 强制规则

- 永远不要跳过 RED 阶段
- 测试必须在实现之前编写
- 覆盖率未达标时不得标记完成
